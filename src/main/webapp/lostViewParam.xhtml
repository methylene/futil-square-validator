This page demonstrates a problem with Stateless JSF. Click on the link above a few times, and the value of the request parameter "barf" will not appear anymore in the headline.

The cause appears to be that, in the case of a SJSF cache hit, all lifecycle phases after RestoreView are skipped.

Commenting out a line in RestoreViewPhase.java "fixes it" for now. I'm not quite sure what other effects this might have. I don't know why the line was there in the first place.

Uncomment the following line in faces-config.xml, to test the fix:


		<!-- 		<lifecycle-factory>org.meth4j.jsf.Meth4jLifecycleFactory</lifecycle-factory> -->

The wstate branch demonstrates the "regular JSF" behaviour of this page.
