Same page as in the "jsfun" branch, but without SJSF. It is intended to show that, when a view contains the f:metadata tag, all 6 lifecycle phases run on an initial request.

Without the following snippet, only phases RESTORE_VIEW (1) and RENDER_RESPONSE (6) would run:


			<f:metadata>
				<f:viewParam name="barf"/>
			</f:metadata>
